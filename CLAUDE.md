# CLAUDE.md

本文件为 Claude Code 在本仓库工作时提供指引。

## 项目定位

- **小丑牌**：Balatro（小丑牌）的网页仿制/改编版。
- 目标：把这款扑克 Roguelike 卡牌游戏在浏览器里跑起来，技术栈倾向 Node / 前端 Web。
- 当前状态：仓库刚初始化，目录为空，尚未选定具体框架（React / Vue / 原生 Canvas 等都待定）。

## 语言

所有回复一律使用 zh-CN，技术术语和代码标识符保持原文。

## 目录约定

仓库还在初始阶段，待技术栈确定后再细化。建议的初始结构：

- `src/`：游戏源码（卡牌、关卡、UI 组件）
- `assets/`：图片、音效、字体等静态资源
- `public/`：直接对外暴露的静态文件（若使用 Vite/Next 这类框架）
- `tests/`：自动化测试

## 常用命令

仓库还没有 `package.json`，下列命令在初始化前端工程后再生效：

- 安装依赖：`npm install`
- 启动开发：`npm run dev`
- 构建产物：`npm run build`
- 跑测试：`npm test`

## 修改时的注意点

- 这是一个游戏项目，状态管理（手牌、回合、计分）会是核心难点，新增模块前先想清楚数据流。
- Balatro 的核心玩法（牌型计分、Joker 效果叠加、商店、Boss Blind）规则复杂，实现一个新机制前先把规则在注释或文档里讲清楚。
- 仅供个人/内部使用，无对外发布、无 LICENSE，但仍建议避免直接搬运 Balatro 原版素材（美术、音乐）以降低风险。
