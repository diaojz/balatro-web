小丑牌 Web · V2.0

Vue 3 + Vite + TypeScript 重构 · 销售牌系统 · 开发执行版 · 2026-05-16

1项目目标

将小丑牌游戏从单文件 HTML 重构为 Vue 3 + Vite + TypeScript 标准项目，实现"打牌计分 → 达分过关 → 商店购买销售牌 → 下一关"的完整循环，并通过销售牌效果叠加让玩家体验指数级分数增长的核心乐趣。

技术约束（V2.0 硬性要求）

  • 框架：Vue 3 Composition API + Vite + TypeScript，不允许选项式 API
  • 状态管理：Pinia，拆分 gameStore（牌局）和 salesStore（销售牌），不允许裸 reactive 跨组件共享
  • 部署：GitHub Pages，CI/CD 走 .github/workflows/deploy.yml（Vite build → gh-pages 分支）
  • 视觉沿用 V1 风格：扑克桌绿 #2d5a3d，橙强调 #e8682a，不引入外部 UI 库

关键 TypeScript Interface

// src/types/index.ts

interface Card {
  id:    string          // 唯一标识，如 "H_A"
  suit:  'H' | 'D' | 'C' | 'S'  // 红心/方块/梅花/黑桃
  rank:  'A' | '2' | ... | 'K'
  value: number         // A=11, J/Q/K=10, 2-10 按面值
}

type HandType =
  'HIGH_CARD' | 'PAIR' | 'TWO_PAIR' | 'THREE_OF_A_KIND' |
  'STRAIGHT' | 'FLUSH' | 'FULL_HOUSE' | 'FOUR_OF_A_KIND' |
  'STRAIGHT_FLUSH' | 'FLUSH_FIVE'

interface HandResult {
  type:         HandType
  baseChips:    number
  baseMult:     number
  scoringCards: Card[]  // 参与计分的牌（计分卡，非踢脚卡）
}

interface ScoreContext {
  handType:     HandType
  scoringCards: Card[]
  allCards:     Card[]  // 本次出的全部牌
}

interface SalesCardDef {
  id:        string
  name:      string
  desc:      string
  price:     number
  sellValue: number          // = Math.floor(price / 2)
  rarity:    'common' | 'rare'
  effect:    SalesCardEffect
}

// 效果函数：接收计分上下文，返回本张牌贡献的 chips/mult/xMult
type SalesCardEffect = (ctx: ScoreContext) => {
  addChips?: number
  addMult?:  number
  xMult?:    number
}

4计分管道详解

计分公式

最终得分 = (hand_chips + Σ scoring_card_chips + Σ addChips) × (hand_mult + Σ addMult) × Π xMult

触发顺序规则（必须严格按此实现）

  1. 初始化：chips = hand_baseChips；mult = hand_baseMult；xMultProduct = 1
  2. 计分牌加 chips：遍历 scoringCards，chips += card.value（踢脚牌不参与）
  3. 销售牌加成（按槽位 0→4 顺序）：对每张已装备的销售牌调用 effect(ctx)，addChips 加到 chips，addMult 加到 mult，xMult 累乘到 xMultProduct
  4. 最终得分：total = Math.floor(chips × mult × xMultProduct)

关键约束：所有 addChips 和 addMult 是加法先做，所有 xMult 是乘法最后做。不允许在遍历销售牌过程中中途做 xMult。

计算例子（开发对照实现用）

例 1：基础场景——一对 5，装备"怒火"(+4 Mult) 和"雕琢"(+30 Chips)，无 xMult
手牌：5♥ 5♦ 7♠ K♣ 3♥，scoringCards = 5♥ 5♦（踢脚牌 7♠ K♣ 3♥ 不加 chips）
hand_baseChips = 10，hand_baseMult = 2
scoringCard chips：5 + 5 = 10
chips 初值 = 10 + 10 = 20
销售牌"怒火"（槽位 0）：addMult=4，mult = 2 + 4 = 6
销售牌"雕琢"（槽位 1）：addChips=30，chips = 20 + 30 = 50
xMultProduct = 1（无 xMult 销售牌）
得分 = Math.floor(50 × 6 × 1) = 300
例 2：花色加成——同花 5 张红桃，装备"色欲"(每红桃 +3 Mult) 和"玻璃工匠"(同花 ×1.5)
手牌：A♥ K♥ 7♥ 5♥ 3♥（同花，5 张全部参与计分）
hand_baseChips = 35，hand_baseMult = 4
scoringCard chips：11 + 10 + 7 + 5 + 3 = 36
chips 初值 = 35 + 36 = 71
销售牌"色欲"（槽位 0）：allCards 含 5 张红桃 → addMult = 5×3 = 15，mult = 4 + 15 = 19
销售牌"玻璃工匠"（槽位 1）：isFlush → xMult=1.5，xMultProduct = 1×1.5 = 1.5
chips = 71（无 addChips）
小数处理：最终得分用 Math.floor 向下取整
得分 = Math.floor(71 × 19 × 1.5) = Math.floor(2023.5) = 2023
例 3：复杂叠加——葫芦（K♥ K♦ K♣ 4♠ 4♥），装备"怒火"(+4 Mult)、"三角洲"(葫芦 ×2) 和"卡文迪什"(×3)
手牌：K♥ K♦ K♣ 4♠ 4♥（葫芦，5 张全部参与计分）
hand_baseChips = 40，hand_baseMult = 4
scoringCard chips：10 + 10 + 10 + 4 + 4 = 38
chips 初值 = 40 + 38 = 78
销售牌"怒火"（槽位 0）：addMult=4，mult = 4 + 4 = 8
销售牌"三角洲"（槽位 1）：handType === FULL_HOUSE → xMult=2，xMultProduct = 1×2 = 2
销售牌"卡文迪什"（槽位 2）：xMult=3，xMultProduct = 2×3 = 6
chips = 78（无 addChips）
关键约束：addMult 全部加完后，xMult 统一在最后做乘法
得分 = Math.floor(78 × 8 × 6) = 3744

5核心流程与决策记录

玩家视角 7 步循环

1
开局：level=1，targetScore=300，money=0，销售槽全空，洗牌，发 8 张手牌，handsLeft=4，discardsLeft=3，phase='playing'
2
出牌 / 弃牌：玩家点选 1-5 张手牌。点"出牌"触发计分管道，score 增加，补牌到 8 张，handsLeft--。点"弃牌"丢弃所选牌，补牌，discardsLeft--。HUD 实时显示 score / targetScore / handsLeft / discardsLeft / money。
3
计分展示：出牌后 ScoreDetail 显示"牌型名 · X Chips × Y Mult × Z = 得分"，停留约 1.5 秒后消失，不阻塞操作。
4
达分判定：每次出牌后检查 score >= targetScore。达分 → money += 5，phase='shop'，进入商店。handsLeft 归零且未达分 → phase='lose'。
5
商店：展示 3 张随机销售牌（从 20 张池中抽取，Common 概率 80%，Rare 概率 20%，Reroll 时重新抽取）。点销售牌买入（money -= price，写入空闲槽位）。点已装备的销售牌卖出（money += sellValue，槽位清空）。点"Reroll"（money -= 5）重新随机 3 张。点"继续"进入下一关。
6
下一关初始化：level++，targetScore = Math.ceil(上一关 targetScore × 1.5)，重新洗牌，发 8 张，handsLeft=4，discardsLeft=3，score=0，phase='playing'。销售槽保留。
7
失败 / 重开：失败画面显示"已到达第 N 关，本关目标分 M"。点"重新开始"：level=1，targetScore=300，money=0，销售槽清空，重新洗牌，phase='playing'。

关键决策记录（2026-05-16 已拍板）

  • D1 · 稀有度简化为 2 档 → Common / Rare
    原版 4 档（Common / Uncommon / Rare / Legendary）对课堂演示过于复杂。2 档足够体现收集差异，UI 颜色对比（蓝灰 vs 橙红）视觉清晰。
  • D2 · 卖出价格 → 半价向下取整（Math.floor(price / 2)）
    与 Balatro 原版一致。计算规则简单，开发无歧义。$5 价格 → 卖出 $2，不是 $2.5。
  • D3 · 关卡奖励固定 $5 → 去掉剩余手牌/弃牌奖励和利息
    课堂时间有限，固定 $5 让玩家专注于销售牌组合逻辑，减少"要留弃牌次数赚钱"的旁支决策。
  • D4 · 销售牌池 20 张 → 涵盖 +Chips / +Mult / xMult / 条件触发 / 全局加成 5 种效果类型
    20 张在课堂演示时够形成明显组合差异，又不至于稀释商店刷新的期望值。牌池未来可按 salesCards.ts 文件直接扩展，不改计分逻辑。
  • D5 · 没有游戏通关上限 → 目标分无限 × 1.5 递增，失败为唯一终点
    无限递增让玩家自然感受到销售牌叠加的指数级效果，符合 Roguelike 教学目的。通关画面可留 V3 做。
