2.4 Joker 系统

核心规则：标准 5 个 Joker 槽位，持续生效（不需要出牌才触发），按从左到右顺序计算。

稀有度：Common（70%）/ Uncommon（25%）/ Rare（5%）/ Legendary（仅通过特定幽灵牌获得）

效果类型说明代表性 Joker 举例
+Chips（加算筹码） 直接增加该手牌的 chips 数值 Jolly Joker：每有一对，+8 chips
+Mult（加算倍率） 直接增加该手牌的 mult 数值 Joker（基础款）：+4 mult
×Mult（乘算倍率） 对最终倍率做乘法，极其强大 Cavendish：×3 mult（有概率自毁）
条件触发 打出特定牌型/花色/点数时生效 Greedy Joker：每张计分钻石牌 +3 mult
持有触发 手中持有某牌时触发（即使不出） Baron：手持每张 K → ×1.5 mult
复制类 复制其他 Joker 或卡牌效果 Blueprint：复制右侧 Joker 的效果
全局增益 对所有出牌永久生效 Baseball Card：每个 Rare Joker ×1.5 mult

来源：balatrowiki.org · Jokers

2.5 计分公式（原版标准）

最终得分 = (hand_chips + Σ scoring_card_chips + Σ joker_chips) × (hand_mult + Σ joker_mult) × Π joker_xmult

其中：

  • hand_chips：当前牌型的基础 chips（受 Planet 牌升级影响）
  • scoring_card_chips：参与计分的每张牌自带的 chips（2-9 按面值，10/J/Q/K=10，A=11）
  • hand_mult：当前牌型的基础 mult（受 Planet 牌升级影响）
  • joker_chips / joker_mult：各 Joker 提供的加算 chips 和 mult
  • joker_xmult：各 Joker 提供的乘算倍率（如 ×3、×1.5）
注意：V1 的公式是 (base + 手牌点数之和) × mult，实际上已经包含了 scoring_card_chips 的概念。V2 只需把这部分拆开、并加入 Joker 的 chips/mult 插槽即可，底层逻辑改动量不大。

4V2.1+ 暂缓清单

以下功能确认不在 V2.0 做，理由附后，后续版本再评估。

  • 消耗牌系统（塔罗牌 + 星球牌 + 幽灵牌）——理由：三种消耗品各自有独立的使用 UI（使用时需要选中目标牌）、存储槽（消耗品槽）和触发时机，工程量独立且复杂。V2.0 Joker 系统已能提供足够的数值增益，先验证主循环再加消耗品。
  • 牌型升级 / 星球牌（Planet Cards）——理由：依赖 handLevels 系统和消耗品购买链路，等消耗牌系统 UI 确定后一起做，否则两次重构 calcScore()。
  • 增强牌（Enhancements）/ 印章（Seals）/ 版本（Editions）——理由：需要在 Card 结构上追加字段，并在 createCardEl() 渲染层和 calcScore() 计分层同时改动；系统互相依赖（塔罗牌是主要施加途径），建议跟消耗牌一起做。
  • Voucher（优惠券）——理由：14 对共 28 种永久效果，大量涉及商店打折、利息上调、槽位扩展等全局状态，单独是一个较大功能块，V2.0 商店先只做 Joker 购买。
  • Booster Pack（补充包）——理由：开包 UI 交互独立，涉及从牌库随机抽消耗品或 Joker，需要先把消耗品系统建好才有意义。
  • Tag（跳过盲注奖励）——理由：V2.0 跳过 Small/Big 的设计里没有 Tag 奖励，先验证跳过流程本身。Tag 种类多（40+），需要独立设计。
  • 更多 Joker（150 张原版 Joker 中的剩余 142 张）——理由：V2.0 先用 8 张验证效果框架，后续按玩法需要分批扩充，不一次全做。
  • 更多 Boss Blind（原版 38 种中的剩余 36 种）——理由：V2.0 实现 2 个足够验证 Boss 机制，后续批量扩展，每个只需在 bossBlinds[] 数组里追加配置。
  • Ante 4–8 / Endless 模式——理由：V2.0 先做 Ante 1–3，验证整体循环手感和平衡性，稳定后才有意义扩展关卡长度。
  • 动画 / 音效 / 移动端适配——继承自 V1 的"不做"列表，长期暂缓。

5.5 Ante 分数常量表

// 分数阈值常量，按 §2.2 数据
const ANTE_BASE = { 1: 300, 2: 800, 3: 2000 };
const BLIND_MULT = { small: 1, big: 1.5, boss: 2 };

function getTargetScore(ante, blind) {
  return Math.floor(ANTE_BASE[ante] * BLIND_MULT[blind]);
}

5.6 计分管道（伪代码）

function calcScore(selectedCards) {
  const handType = evaluateHand(selectedCards);
  const level = state.handLevels[handType.key];

  // 1. 基础 chips = 牌型chips + 各计分牌面值
  const scoringCards = getScoringCards(selectedCards, handType);
  let chips = handType.baseChips(level);
  for (const c of scoringCards) {
    if (!c.debuffed) chips += c.value; // 被 debuff 的牌不贡献 chips
  }

  // 2. 基础 mult
  let mult = handType.baseMult(level);

  // 3. 各 Joker 加算 chips 和 mult
  let xMultProduct = 1;
  const ctx = { hand: state.hand, scoringCards, handType, chips, mult };
  for (const joker of state.jokers) {
    const { addChips = 0, addMult = 0, xMult = 1 } = joker.effect(ctx);
    chips += addChips;
    mult += addMult;
    xMultProduct *= xMult;
  }

  // 4. 最终得分
  const finalScore = Math.floor(chips * mult * xMultProduct);
  return { handType, chips, mult, xMultProduct, score: finalScore };
}

6核心游戏流程（V2.0）

完整一局流程

游戏开始

  1. 初始化：ante=1, money=$0, jokers=[], 洗牌，phase='blind_select'
  2. 盲注选择界面（phase=blind_select）：显示 Small/Big/Boss 分数要求，玩家选择打或跳过（Small/Big）
  3. 打牌阶段（phase=playing）：发 8 张手牌，设置 targetScore，玩家出牌/弃牌，达到 targetScore 则本 Blind 胜利
  4. 结算 Toast：显示本轮奖励明细，money += 奖励+利息，持续 2–3 秒
  5. 商店（phase=shop）：显示 2 个随机 Joker + 刷新 + 继续按钮，玩家购买或直接继续
  6. 若 3 个 Blind 均通过 → ante++，返回步骤 2（blind_select）
  7. ante 推进到 4（即打完 ante=3 的 Boss Blind）→ phase='win'，游戏胜利
  8. 任意 Blind 内出牌耗尽且 score < targetScore → phase='lose'，显示进度

风险说明建议对策

开放问题决策记录（2026-05-16 用户拍板）

  • Q1 · Joker 数量？ → 维持 8 张
    V2.0 底池固定 8 张 Joker，商店每次随机展示 2 张。可控、能尽快跑通闭环；重复度高的问题留到 V2.1+ 扩库时解决。
  • Q2 · 跳过 Blind 的奖励？ → 不给奖励
    跳过 Small/Big 不给任何金钱或 Tag。设计上"跳过"等同于"放弃这场的金钱与利息收益换取节省时间"，是一种纯权衡，而非纯收益。Tag 系统整体顺延到 V2.1+。
  • Q3 · 商店第 3 格？ → 加 Coming Soon 占位格
    商店布局变为「2 个 Joker 格 + 1 个塔罗/星球牌占位格」。占位格显示一张随机的塔罗或星球牌外观，标注 "Coming Soon"，不可购买、不可交互。仅用于：① 让商店视觉更饱满；② 给玩家对 V2.1+ 的期待。开发时可以做成纯展示组件，不接计分逻辑。
