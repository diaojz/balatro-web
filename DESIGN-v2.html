稀有度选色逻辑说明

  • Common 蓝灰 #6b7db3：饱和度低，与桌面绿的对比适中，不抢眼，符合"普通"的心理预期。
  • Uncommon 绿 #4ade80：与桌面底色 #2d5a3d 同色系但更亮，有辨识度又不突兀。
  • Rare 红 #f87171：使用亮红（区别于弃牌红 #a83232），更接近"珍贵"而非"危险"的语义。
  • Legendary 紫 #a855f7：游戏中未使用过的色相，视觉上"稀缺"，配合脉冲动画强化神秘感。
与 Boss 红的区分：Rare 用 #f87171（亮桃红），Boss 也用 #f87171，两者相同但语义不同。实际使用时 Boss Blind 有独立的背景色 #2e1a1a + 发光阴影来区分场景，不会造成混淆——玩家不会同时看到 Rare Joker 和 Boss 边框。

"跳过"按钮语义设计

PRD Q2 决策：跳过 Small/Big 不给任何奖励。按钮文案必须明确告知玩家"无奖励"这件事，而非含糊的"跳过"。

按钮类型文案样式
Small/Big 打 "打 →" #e8682a 橙色实心，全宽
Small/Big 跳过 "跳过（无奖励）" 透明背景，白色半透明边框
文字 rgba(255,255,255,.5)
Boss 必须打 "必须打 →" #f87171 红色实心，全宽
Boss 不可跳过 "Boss 不可跳过" opacity: .25，cursor: not-allowed，disabled
跳过时的反馈：点击跳过后，该盲注卡片灰化（opacity 降至 .35）并显示"跳过 · 无奖励"灰色文字，然后直接进入商店，不弹额外提示。灰化是玩家"错过了什么"的视觉反馈。

10 · 开发实现交班
给工程师的实现建议，对应 PRD §5 数据结构与 §6 布局建议。

CSS 变量追加（copy-paste 到 :root）

  • 新增 14 个变量，见第 02 节完整列表，直接追加到现有 :root {} 末尾
  • 稀有度颜色以 --color-rarity-[档位] 命名，便于 JS 动态读取：getComputedStyle(el).getPropertyValue('--color-rarity-' + joker.rarity)
  • Joker 背景色以 --joker-bg-[档位] 命名，在 createJokerEl() 中按 rarity 赋值给 style.background

Joker 卡片 HTML 结构建议

  • 建议结构：<div class="joker-card" data-rarity="common" data-id="joker_base">
  • 内部四个子节点：.j-icon（emoji）、.j-name、.j-effect、.j-price
  • 稀有度边框通过 data-rarity + CSS 属性选择器 [data-rarity="uncommon"] 设置 box-shadow，不要用 JS 内联
  • 商店 Joker（90×120px）加 class joker-card--shop；槽位 Joker（70×100px）加 joker-card--slot
  • 槽位已满时在 .joker-slot-area 上加 class is-full，CSS 联动禁止 pointer-events

布局改动（V1 → V2）

  • V1 的 .game-viewport 保持 1080×640，内部 flex-direction: column
  • HUD 下方新增 .game-body 容器，display: flex，flex-direction: row
  • .joker-slot-area 宽 110px，flex-shrink: 0，在 .game-body 左侧
  • .main-area flex: 1，包含原 V1 的出牌区 + 手牌区 + 操作栏
  • 手牌 8 张在 970px 内横排，每张 82×118px，gap 约 9px，合计约 746px，有余量

phase 切换逻辑与视图渲染

  • state.phase 变化时，调用 renderPhase() 函数，根据 phase 值显示/隐藏对应 DOM 区域
  • 建议用 CSS class 控制可见性：.phase-[name] { display: flex }，配合 body.phase-playing 类名切换
  • blind_select / shop 全屏覆盖：position absolute，z-index 10，覆盖 .game-body，但 HUD 仍可见（HUD z-index 5）
  • Boss 提示卡 z-index 20，遮住整个 game-viewport，3 秒后 JS removeEventListener + classList.remove

计分显示（F1 改动）

  • V1 的 score-detail 区域改为三段显示：Chips [135] × Mult [9] = [1215]
  • chips 数字颜色 #5dd67a（沿用 score 绿），mult 数字颜色 #e8682a（橙），×Mult 数字颜色 #a855f7（紫）
  • 若 joker_xmult = 1（无乘算），×Mult 段不显示，避免界面冗余
  • 数字跳动动画：requestAnimationFrame 在 400ms 内从旧值线性插值到新值，font-variant-numeric: tabular-nums 防抖

Coming Soon 占位实现建议

  • 用纯 CSS + HTML 实现，不需要任何 JS 逻辑
  • pointer-events: none 写在 CSS 里，JS 无需额外判断
  • V2.1 接入塔罗牌后，只需移除 coming-soon class 并补充点击事件，零额外改动
  • emoji 建议从固定列表中随机抽取显示（🔮 🌍 ☀️ 🌙 ⭐），增加视觉多样性但不影响功能

Joker 效果计算顺序（F4 关键）

  • 严格按 PRD §5.6 伪代码顺序：先累计所有 addChips → 再累计所有 addMult → 最后乘 xMultProduct
  • Cavendish 的 xMult: 3 必须在 for 循环最末处理，用 xMultProduct *= xMult
  • Joker 顺序影响 ×Mult 结果：state.jokers 数组顺序即为计算顺序，槽位从左到右
  • Joker 触发高亮动画：在 for 循环中，若某 Joker 返回非零值则 triggerJokerAnim(joker.id)
设计规范总结

V2.0 在"克制·专业·带赌场感"的 V1 气质上，新增了稀有度层次感这一核心维度——通过 4 档颜色边框（蓝灰/绿/红/紫）让玩家在 0.5 秒内识别 Joker 价值，这是 Balatro 原版最具辨识度的视觉语言。 主色仍是扑克桌绿 #2d5a3d + 强调橙 #e8682a，未做任何改变。

最值得开发前 review 的 2 个设计点：

  • ① Joker 稀有度配色：Rare 红（#f87171）与 Boss 警示红（#f87171）使用了相同 hex 值，但所处上下文不同（一个在 Joker 卡，一个在盲注选择 / Boss 提示卡），请确认这种"同色不同语义"在实际界面中不会造成视觉混淆。如果认为有问题，可以把 Rare 边框调整为 #fb923c（橙红）以作区分。
  • ② Coming Soon 占位格视觉权重：当前方案为 opacity .65 + 灰化 emoji + 虚线蓝边，整体视觉权重约为可购买格的 40%。请评估：在商店只有 3 格的情况下，这个占位是否过于显眼（影响玩家焦点），还是恰好合适（传递"即将上线"期待感）。
